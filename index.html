Dokumentasi H2H

Pengenalan

Selamat datang di dokumentasi REST API H2H. Dokumentasi ini menyediakan panduan lengkap untuk mengintegrasikan layanan API ke dalam aplikasi Anda dengan mudah dan cepat. API ini mendukung berbagai transaksi seperti pembelian pulsa, paket data, pembayaran tagihan, dan banyak lagi.

Base URL:

https://demo-h2h.serpul.co.id
https://member-api.demo-h2h.serpul.co.id

Silahkan sesuaikan base URL sesuai domain admin masing-masing.

  • Contoh base URL: https://demo-h2h.serpul.co.id
  • Contoh base URL API member: https://member-api.demo-h2h.serpul.co.id

Catatan Penting

Transaksi hanya dapat dilakukan oleh IP customer yang sudah di-whitelist. Untuk callback, whitelist IP ditambahkan di fitur multivendor.

Layanan API yang Tersedia

  • Transaksi Prabayar - Untuk melakukan pembelian produk prabayar seperti pulsa, paket data, voucher game, dll.
  • Pengecekan Tagihan Pascabayar - Untuk memeriksa detail tagihan seperti PLN, PDAM, BPJS, dll.
  • Pembayaran Tagihan Pascabayar - Untuk melakukan pembayaran tagihan yang sudah dicek sebelumnya.
  • Callback Transaksi - Untuk menerima notifikasi status transaksi secara real-time.

Format Kode Produk

  • Untuk pengecekan tagihan, tambahkan awalan C pada kode produk. Contoh: untuk tagihan PLN dengan kode produk PLN, gunakan CPLN pada parameter product.
  • Untuk pembayaran tagihan, tambahkan awalan B pada kode produk. Contoh: untuk pembayaran PLN dengan kode produk PLN, gunakan BPLN pada parameter product.

Catatan Penting

  • Untuk mengakses endpoint data produk, Anda memerlukan token autentikasi yang valid.
  • Pastikan untuk menyertakan header Authorization dengan format "Bearer token" pada setiap request.

Callback Transaksi

Endpoint ini digunakan untuk menerima callback dari supplier ke H2H.

Semua nama parameter bersifat opsional contoh refid bisa diganti ke idtrx, clientid, idtransaction, dll. Message bisa diganti ke note, description, dll. Namun callback yang dikirimkan harus memiliki komponen sebagai berikut: id transaksi h2h, harga, pesan sukses, saldo baik tergabung dalam 1 parameter ataupun sebagai parameter terpisah.

GET https://demo-h2h.serpul.co.id/callback/transaction

Parameter

Parameter Tipe Deskripsi Contoh
refid String ID transaksi H2H (boleh dimasukkan di message) PRA5603cb5ea2f58780a
price Integer Harga produk (boleh dimasukkan di message) 49300
message String Pesan callback R#PRA5603cb5ea2f58780aT#65664602SRVP50...

Contoh Request

GET https://demo-h2h.serpul.co.id/callback/transaction?refid=PRA5603cb5ea2f58780a&price=49300&message=R%23PRA5603cb5ea2f58780aT%2365664602SRVP50.082258428838SUKSES.SN/Ref:03730600000375025136.Saldo2.776.640-49.300=2.727.340@19/0313.46.25C%231KP%23SRVP50.
